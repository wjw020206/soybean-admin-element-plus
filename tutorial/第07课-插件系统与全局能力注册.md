# 第 7 课：插件系统与全局能力注册

## 课程定位

第 5 课从 `src/main.ts` 理解了应用启动链路，第 6 课理解了 Vite 配置和环境变量。第 7 课继续沿着启动链路往下看：项目中的全局插件和全局能力是如何集中注册的。

本课重点不是 Vite 插件，而是应用运行时插件和全局初始化逻辑。它们主要集中在 `src/plugins` 目录中，包括全局样式、Element Plus、首屏 loading、NProgress、Iconify、Dayjs 和版本更新检测。

学完本课后，你应该能知道每个 `setupXxx` 函数在页面中产生什么效果，也能理解为什么项目要把这些全局能力集中放在 `src/plugins` 中。

## 学习目标

学完本课后，你应该能够：

- 说明 `src/plugins` 目录的职责。
- 理解 `src/plugins/index.ts` 为什么只做统一导出。
- 理解 `assets.ts` 中的副作用导入。
- 理解 `setupUI` 如何注册 Element Plus 和修改默认 props。
- 理解 `setupLoading` 如何在 Vue 挂载前显示首屏 loading。
- 理解 `setupNProgress` 如何配合路由守卫显示顶部进度条。
- 理解 `setupIconifyOffline` 如何配置 Iconify 图标服务。
- 理解 `setupDayjs` 如何初始化日期库和语言。
- 理解 `setupAppVersionNotification` 如何做应用版本更新检测。
- 能够尝试关闭某个插件并观察页面影响。

## 一、`src/plugins` 目录是做什么的

当前目录结构：

```txt
src/plugins
├── app.ts
├── assets.ts
├── dayjs.ts
├── iconify.ts
├── index.ts
├── loading.ts
├── nprogress.ts
└── ui.ts
```

它的职责是：**集中管理应用级初始化逻辑**。

这些逻辑有几个共同特点：

- 不是某个页面独有。
- 通常在应用启动时执行。
- 影响整个应用。
- 有些需要注册到 Vue app。
- 有些不依赖 Vue app，只需要执行一次。

例如：

- Element Plus 注册后，整个项目都能使用 Element Plus 组件。
- NProgress 注册后，路由切换能显示进度条。
- Dayjs 初始化后，日期语言和插件全局生效。
- 全局样式导入后，所有页面样式基础一致。

所以它们不适合散落在页面组件里，而是集中放在 `src/plugins`。

## 二、`src/plugins/index.ts`：统一导出入口

当前内容：

```ts
export * from './loading';
export * from './nprogress';
export * from './iconify';
export * from './dayjs';
export * from './app';
export * from './ui';
```

它本身不做复杂逻辑，只是统一导出。

这样 `src/main.ts` 可以写：

```ts
import {
  setupAppVersionNotification,
  setupDayjs,
  setupIconifyOffline,
  setupLoading,
  setupNProgress,
  setupUI
} from './plugins';
```

而不是写成：

```ts
import { setupLoading } from './plugins/loading';
import { setupNProgress } from './plugins/nprogress';
import { setupIconifyOffline } from './plugins/iconify';
import { setupDayjs } from './plugins/dayjs';
import { setupUI } from './plugins/ui';
import { setupAppVersionNotification } from './plugins/app';
```

这是一种 barrel export 写法。

优点：

- `main.ts` 导入更简洁。
- 插件入口集中。
- 新增插件时只需要在 `index.ts` 导出。

注意：`index.ts` 只是导出，不代表所有插件都会自动执行。真正执行是在 `main.ts` 中调用对应函数。

## 三、`assets.ts`：全局资源副作用导入

当前内容：

```ts
import 'virtual:svg-icons-register';
import 'element-plus/dist/index.css';
import 'element-plus/theme-chalk/dark/css-vars.css';
import 'uno.css';
import '../styles/css/global.css';
import 'swiper/css';
import 'swiper/css/navigation';
import 'swiper/css/pagination';
```

这个文件没有导出任何变量或函数。它的作用是：**执行这些导入，让全局资源生效**。

### 1. `virtual:svg-icons-register`

这是 `vite-plugin-svg-icons` 提供的虚拟模块。

作用：

- 扫描本地 SVG 图标。
- 注册 SVG sprite。
- 让项目可以使用本地图标。

相关目录：

```txt
src/assets/svg-icon
```

### 2. `element-plus/dist/index.css`

Element Plus 基础样式。

如果去掉，Element Plus 组件可能只剩结构，没有正确视觉样式。

### 3. `element-plus/theme-chalk/dark/css-vars.css`

Element Plus 暗色模式 CSS 变量。

它配合项目的 `html.dark` 暗色 class 使用。

### 4. `uno.css`

UnoCSS 虚拟 CSS 入口。

作用：

- 让 `flex`、`gap-16px`、`bg-layout`、`text-primary` 等原子类生成并生效。

如果去掉，项目中大量 UnoCSS class 会失效。

### 5. `../styles/css/global.css`

项目自己的全局 CSS。

通常包含：

- reset。
- transition。
- nprogress。
- 全局基础样式。

### 6. `swiper/css`

Swiper 插件样式。

项目中使用 Swiper 示例页面时需要这些样式。

## 四、为什么 `assets.ts` 在 `main.ts` 顶部导入

`main.ts` 中：

```ts
import './plugins/assets';
```

这是副作用导入。

它的意义是：

```txt
应用启动时先加载全局资源
```

这些资源不需要等 Vue app 创建，因为它们是全局样式或虚拟模块注册。

如果这些导入放得太晚，可能出现：

- 页面首次渲染时样式缺失。
- 图标无法正常显示。
- 暗色变量未准备好。
- UnoCSS 原子类不生效。

所以全局资源通常在入口文件顶部导入。

## 五、`ui.ts`：Element Plus 注册和默认行为调整

当前内容：

```ts
import type { App } from 'vue';
import ElementPlus, { ElCard, ElForm, ElTable } from 'element-plus';

/** global table column align */
ElTable.TableColumn.props.align = {
  type: String,
  default: 'center'
};

/** global ElCard shadow */
ElCard.props.shadow = {
  type: String,
  default: 'never'
};

/** global ElForm require asterisk position */
ElForm.props.requireAsteriskPosition = {
  type: String,
  default: 'right'
};

/** full import ElementPlus */
export const setupUI = (app: App) => {
  app.use(ElementPlus);
};
```

它做两类事情。

### 1. 修改部分 Element Plus 组件默认 props

表格列默认居中：

```ts
ElTable.TableColumn.props.align = {
  type: String,
  default: 'center'
};
```

卡片默认无阴影：

```ts
ElCard.props.shadow = {
  type: String,
  default: 'never'
};
```

表单必填星号默认在右侧：

```ts
ElForm.props.requireAsteriskPosition = {
  type: String,
  default: 'right'
};
```

这种做法适合统一项目默认规范。

例如你不用每个卡片都写：

```vue
<ElCard shadow="never">
```

因为默认已经是 `never`。

### 2. 全量注册 Element Plus

```ts
app.use(ElementPlus);
```

这会把 Element Plus 插件安装到 Vue app 中。

因此项目中可以使用：

```vue
<ElButton />
<ElCard />
<ElTable />
<ElForm />
```

当前项目是全量导入 Element Plus。好处是简单，缺点是体积可能比按需导入更大。

## 六、样式修改和组件封装的边界

在 `ui.ts` 中修改默认 props，在 `styles/scss/element-plus.scss` 中修改全局样式，这属于项目级 UI 规范。

例如：

- 所有 `ElCard` 默认无阴影。
- 所有表格列默认居中。
- 所有卡片 header 字号统一。

这类不需要二次封装组件。

只有当你要表达业务结构时，才考虑封装。

例如：

```vue
<PageCard title="用户管理">
  <template #extra>
    <ElButton>新增</ElButton>
  </template>
</PageCard>
```

总结：

- 改默认视觉：全局样式或默认 props。
- 改业务结构：封装组件。

## 七、`loading.ts`：首屏 loading

`setupLoading()` 在 `createApp(App)` 之前执行。

它做的核心事情是：

```ts
const app = document.getElementById('app');

if (app) {
  app.innerHTML = loading;
}
```

也就是说，在 Vue 真正挂载之前，先往 `#app` 中写一段 loading HTML。

这样用户不会看到空白页面。

### 1. 读取主题色

```ts
const themeColor = localStg.get('themeColor') || '#646cff';
const darkMode = localStg.get('darkMode') || false;
const palette = getColorPalette(themeColor);
```

它会从本地存储中读取：

- 主题色。
- 暗色模式。

这样 loading 的颜色能尽量和用户上一次设置保持一致。

### 2. 注入 CSS 变量

```ts
const primaryColor = `--primary-color: ${r} ${g} ${b}`;

const svgCssVars = Array.from(palette.entries())
  .map(([key, value]) => `--logo-color-${key}: ${value}`)
  .join(';');
```

这些变量用于 loading 中的 logo 和主色。

### 3. 恢复暗色 class

```ts
if (darkMode) {
  toggleHtmlClass(DARK_CLASS).add();
}
```

如果用户上次是暗色模式，loading 阶段就先给 `html` 添加 `dark` class，避免一开始亮一下再变暗。

### 4. 写入 loading HTML

```ts
app.innerHTML = loading;
```

等后面执行：

```ts
app.mount('#app');
```

Vue 会接管 `#app`，loading 被真实应用替换。

## 八、`// @unocss-include` 是什么

`loading.ts` 顶部有：

```ts
// @unocss-include
```

这是给 UnoCSS 的提示。

因为 `loading.ts` 中的 class 是写在字符串模板里的：

```ts
const loading = `
<div class="fixed-center flex-col bg-layout" style="${cssVars}">
...
</div>`;
```

UnoCSS 需要扫描源码中的 class 才能生成对应 CSS。

`@unocss-include` 的作用是告诉 UnoCSS：这个文件中也包含需要扫描的原子类。

如果没有它，某些写在字符串里的 class 可能不会被识别，loading 样式可能缺失。

## 九、`nprogress.ts`：顶部进度条

当前内容：

```ts
import NProgress from 'nprogress';

/** Setup plugin NProgress */
export function setupNProgress() {
  NProgress.configure({ easing: 'ease', speed: 500 });

  // mount on window
  window.NProgress = NProgress;
}
```

它做两件事：

1. 配置 NProgress 动画参数。
2. 把 NProgress 挂到 `window`。

为什么挂到 `window`？

因为路由守卫中会使用：

```ts
window.NProgress?.start?.();
window.NProgress?.done?.();
```

对应文件：

```txt
src/router/guard/progress.ts
```

流程是：

```txt
setupNProgress()
→ window.NProgress = NProgress
→ 路由 beforeEach 开始进度条
→ 路由 afterEach 结束进度条
```

如果关闭 `setupNProgress()`，页面仍能跳转，但顶部进度条不会出现。

## 十、`iconify.ts`：Iconify 图标服务

当前内容：

```ts
import { addAPIProvider } from '@iconify/vue';

/** Setup the iconify offline */
export function setupIconifyOffline() {
  const { VITE_ICONIFY_URL } = import.meta.env;

  if (VITE_ICONIFY_URL) {
    addAPIProvider('', { resources: [VITE_ICONIFY_URL] });
  }
}
```

它用于配置 Iconify 的图标 API 地址。

默认情况下，Iconify 可能从公共服务加载图标。如果企业内网无法访问外网，可以设置：

```bash
VITE_ICONIFY_URL=你的内网图标服务地址
```

然后项目会使用这个地址作为 Iconify provider。

如果没有配置 `VITE_ICONIFY_URL`，函数不会做任何事。

## 十一、`dayjs.ts`：日期库初始化

当前内容：

```ts
import { extend } from 'dayjs';
import localeData from 'dayjs/plugin/localeData';
import { setDayjsLocale } from '../locales/dayjs';

export function setupDayjs() {
  extend(localeData);

  setDayjsLocale();
}
```

它做两件事：

1. 扩展 Dayjs 插件 `localeData`。
2. 根据当前语言设置 Dayjs locale。

为什么要初始化 Dayjs？

后台系统中经常要展示：

- 创建时间。
- 更新时间。
- 操作时间。
- 图表时间。
- 日志时间。

日期库如果没有统一初始化，可能出现语言、格式、周起始日等不一致。

## 十二、`app.ts`：应用级增强能力

`app.ts` 里有两个函数：

```ts
setupAppErrorHandle(app)
setupAppVersionNotification()
```

### 1. `setupAppErrorHandle`

当前函数：

```ts
export function setupAppErrorHandle(app: App) {
  app.config.errorHandler = (err, vm, info) => {
    console.error(err, vm, info);
  };
}
```

它用于配置 Vue 全局错误处理。

不过当前 `src/main.ts` 没有调用它：

```ts
setupAppErrorHandle(app);
```

所以它是预留能力。

如果你想启用全局错误捕获，可以在 `main.ts` 中 `createApp` 后调用。

### 2. `setupAppVersionNotification`

这个函数用于检查应用是否有新版本。

触发条件：

```ts
const canAutoUpdateApp = import.meta.env.VITE_AUTOMATICALLY_DETECT_UPDATE === 'Y' && import.meta.env.PROD;
```

也就是说：

- 环境变量 `VITE_AUTOMATICALLY_DETECT_UPDATE=Y`
- 并且当前是生产环境

才会启用。

## 十三、版本更新检测如何工作

构建时，项目会把构建时间注入两个地方。

### 1. 注入到 JS 常量

`vite.config.ts` 中：

```ts
define: {
  BUILD_TIME: JSON.stringify(buildTime)
}
```

前端代码中可以直接访问：

```ts
BUILD_TIME
```

### 2. 注入到 HTML meta

`build/plugins/html.ts` 中：

```ts
return html.replace('<head>', `<head>\n    <meta name="buildTime" content="${buildTime}">`);
```

构建后的 `index.html` 会包含：

```html
<meta name="buildTime" content="2026-04-23 12:00:00">
```

### 3. 定时拉取最新 index.html

`setupAppVersionNotification()` 会定时请求：

```ts
fetch(`${baseUrl}index.html?time=${Date.now()}`)
```

加 `time=${Date.now()}` 是为了避免浏览器缓存。

### 4. 比较构建时间

它会读取最新 HTML 中的：

```html
<meta name="buildTime" content="...">
```

然后和当前 JS 中的：

```ts
BUILD_TIME
```

比较。

如果不一样，说明服务器已经部署了新版本，但用户浏览器还在运行旧版本 JS。

此时显示通知，让用户刷新页面。

## 十四、为什么版本检测只在生产环境开启

开发环境下，Vite 本身有 HMR。

你修改代码后，开发服务器会自动更新页面，不需要版本更新提示。

版本检测主要解决生产环境问题：

```txt
用户打开页面很久不刷新
服务器已经发布新版本
用户仍然运行旧 JS
可能出现接口或资源不匹配
```

所以它只在生产环境启用。

## 十七、为什么集中注册插件

集中注册有几个好处：

### 1. 启动流程清晰

你只需要看 `main.ts`，就知道应用启动时初始化了哪些全局能力。

### 2. 降低页面耦合

页面组件不需要关心：

- NProgress 怎么初始化。
- Dayjs 插件怎么扩展。
- Element Plus 默认 props 在哪里改。
- 全局样式在哪里导入。

### 3. 方便统一维护

比如以后要换 UI 库，或调整 Element Plus 默认配置，优先看：

```txt
src/plugins/ui.ts
```

### 4. 方便排查问题

比如图标不显示，可以先看：

```txt
src/plugins/assets.ts
src/plugins/iconify.ts
```

比如顶部进度条不显示，可以看：

```txt
src/plugins/nprogress.ts
src/router/guard/progress.ts
```

## 十八、插件和 Vite 插件不要混淆

第 6 课讲过 Vite 插件，例如：

- `@vitejs/plugin-vue`
- `@unocss/vite`
- `vite-plugin-svg-icons`
- `vite-plugin-vue-devtools`

这些是构建工具层面的插件，配置在：

```txt
build/plugins
vite.config.ts
```

第 7 课讲的是应用运行时插件，例如：

- Element Plus。
- NProgress。
- Dayjs。
- Iconify。
- Loading。
- 版本更新检测。

它们配置在：

```txt
src/plugins
```

简单区分：

```txt
Vite 插件：让项目能被开发和构建
应用插件：让应用运行时具备全局能力
```

### 演示 1：从 `main.ts` 找到插件入口

打开 `src/main.ts`，指出：
