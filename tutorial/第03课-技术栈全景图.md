# 第 3 课：技术栈全景图

## 课程定位

第 2 课解决了“项目如何跑起来”的问题，第 3 课开始回答“这个项目由哪些技术组成，以及每个技术负责什么”。

学习一个后台模板时，不能只记住它用了 Vue、Vite、Element Plus。更重要的是理解这些技术之间的分工：谁负责页面渲染，谁负责构建，谁负责路由，谁负责状态，谁负责样式，谁负责请求，谁负责类型检查，谁负责工程规范。

本课不会深入每个技术的全部细节，而是建立一张技术栈全景图。后续课程会沿着这张图逐个深入。

## 学习目标

学完本课后，你应该能够：

- 从 `package.json` 中识别项目核心技术栈。
- 说明 Vue 3、Vite、TypeScript、Vue Router、Pinia、Element Plus、UnoCSS 在项目中的职责。
- 区分运行时依赖 `dependencies` 和开发时依赖 `devDependencies`。
- 理解 Axios / Alova、vue-i18n、Elegant Router、Iconify、Dayjs 等辅助技术的作用。
- 知道哪些技术是后台项目必须掌握的主线，哪些是后续增强能力。
- 建立“技术职责分层”的思维，而不是只背依赖名称。

## 建议课时

建议用 60-90 分钟完成本课：

- 15 分钟：阅读 `package.json`，识别核心依赖。
- 20 分钟：理解主技术栈的职责分工。
- 20 分钟：对照 `src/main.ts` 看技术如何被注册。
- 15 分钟：阅读 `vite.config.ts` 和 `uno.config.ts`。
- 10 分钟：完成技术栈说明表。

## 三、Vue 3：应用主体框架

Vue 3 是这个项目的应用框架，负责：

- 组件化开发。
- 响应式数据。
- 模板渲染。
- 组合式 API。
- 生命周期。
- 页面和组件之间的数据组织。

项目入口在 `src/main.ts`：

```ts
import { createApp } from 'vue';
import App from './App.vue';

const app = createApp(App);
app.mount('#app');
```

实际项目中间还注册了 UI、store、router、i18n 等能力：

```ts
setupUI(app);
setupStore(app);
await setupRouter(app);
setupI18n(app);
```

可以这样理解：

- Vue 负责把组件渲染成页面。
- 其他库通过插件或函数接入 Vue 应用。
- 后台系统中的页面、表格、表单、弹窗、布局本质上都是 Vue 组件。

学习重点：

- `<script setup>`。
- `ref`、`reactive`、`computed`。
- `defineProps`、`defineEmits`。
- 组件拆分。
- 组合式函数 hooks。

## 四、Vite / rolldown-vite：开发和构建工具

Vite 负责项目的开发服务器和生产构建。

本项目的 `package.json` 中：

```json
{
  "vite": "npm:rolldown-vite@7.3.1"
}
```

这表示项目使用的是 `rolldown-vite`，但包名别名为 `vite`。你在脚本中仍然通过 `vite` 命令启动：

```json
{
  "dev": "vite --mode test",
  "build": "vite build --mode prod",
  "preview": "vite preview"
}
```

Vite 在本项目中负责：

- 启动开发服务器。
- 支持热更新。
- 读取环境变量。
- 配置路径别名。
- 配置代理。
- 加载 Vue 插件。
- 加载 UnoCSS 插件。
- 执行生产构建。

重点配置在 `vite.config.ts`：

```ts
export default defineConfig(configEnv => {
  const viteEnv = loadEnv(configEnv.mode, process.cwd()) as unknown as Env.ImportMeta;

  return {
    base: viteEnv.VITE_BASE_URL,
    resolve: {
      alias: {
        '~': fileURLToPath(new URL('./', import.meta.url)),
        '@': fileURLToPath(new URL('./src', import.meta.url))
      }
    },
    server: {
      host: '0.0.0.0',
      port: 9527,
      open: true,
      proxy: createViteProxy(viteEnv, enableProxy)
    }
  };
});
```

学习重点：

- `defineConfig`。
- `loadEnv`。
- `resolve.alias`。
- `server.port`。
- `server.proxy`。
- `plugins`。
- `build`。

## 五、TypeScript：类型系统

TypeScript 给 JavaScript 增加类型系统。

在后台项目里，TypeScript 非常重要，因为后台系统有大量接口、表格字段、表单字段、路由配置和权限数据。如果没有类型约束，字段改名、接口变更、参数传错都很容易在运行时才暴露。

TypeScript 在本项目中负责：

- 约束接口请求参数。
- 约束接口响应数据。
- 约束用户信息、角色、菜单等业务类型。
- 约束路由名称和路径。
- 约束组件 props 和事件。
- 提供编辑器智能提示。

本项目提供类型检查命令：

```json
{
  "typecheck": "vue-tsc --noEmit --skipLibCheck"
}
```

含义：

- `vue-tsc`：检查 Vue + TypeScript 项目。
- `--noEmit`：只检查类型，不生成文件。
- `--skipLibCheck`：跳过第三方库声明文件检查。

对应源码：

- `tsconfig.json`
- `src/typings`
- `src/typings/api`
- `src/router/elegant`

学习重点：

- 基础类型。
- 接口类型。
- 泛型。
- 类型命名空间。
- Vue 组件类型。
- API 类型定义。

## 六、Vue Router：页面路由和访问控制

Vue Router 负责前端页面路由。

在普通项目里，路由主要负责页面跳转。在后台项目里，路由还承担更多职责：

- 登录访问控制。

- 面包屑生成。
- 标签页管理。
- 页面缓存。
- 页面标题。
- 外链处理。

本项目路由入口在 `src/router/index.ts`：

```ts
export const router = createRouter({
  history: historyCreatorMap[VITE_ROUTER_HISTORY_MODE](VITE_BASE_URL),

```

```ts
export async function setupRouter(app: App) {
  app.use(router);
  createRouterGuard(router);
```

注意：项目启动时先注册内置路由，然后在路由守卫和 route store 中初始化常量路由、权限路由。

对应源码：

- `src/router/index.ts`

学习重点：

## 九、UnoCSS：原子化样式系统

UnoCSS 是原子化 CSS 引擎。

项目中经常能看到这样的 class：

```vue
<div class="min-h-500px flex-col-stretch gap-16px overflow-hidden">
```

这些不是传统手写 CSS 类，而是 UnoCSS 根据 class 名动态生成的样式。

本项目配置在 `uno.config.ts`：

```ts
export default defineConfig<Theme>({
  theme: {
    ...themeVars,
    fontSize: {
      'icon-xs': '0.875rem',
      'icon-small': '1rem',
      icon: '1.125rem',
      'icon-large': '1.5rem',
      'icon-xl': '2rem'
    }
  },
  shortcuts: {
    'card-wrapper': 'rd-8px shadow-sm'
  },
  transformers: [transformerDirectives(), transformerVariantGroup()],
  presets: [presetWind3({ dark: 'class' }), presetSoybeanAdmin()]
});
```

几个重点：

- `presetWind3` 提供类似 Tailwind / Windi 的原子类能力。
- `presetSoybeanAdmin` 是项目自己的 UnoCSS 预设。
- `shortcuts` 定义快捷类，例如 `card-wrapper`。
- `themeVars` 把项目主题变量接入 UnoCSS。

学习重点：

- 原子类基本写法。
- 响应式前缀。
- 暗色模式。
- shortcuts。
- 主题变量。
- 与 SCSS 的分工。

## 十四、dependencies 和 devDependencies 的区别

`package.json` 中依赖分为：

- `dependencies`
- `devDependencies`

### dependencies

运行时依赖。

这些依赖通常会参与应用运行或构建后的业务代码。

例如：

- `vue`
- `vue-router`
- `pinia`
- `element-plus`
- `vue-i18n`
- `dayjs`
- `echarts`
- `@sa/axios`
- `@sa/hooks`

如果没有这些依赖，应用运行时功能可能直接不可用。

### devDependencies

开发时依赖。

这些依赖主要用于开发、构建、检查、生成代码。

例如：

- `typescript`
- `vue-tsc`
- `eslint`
- `@vitejs/plugin-vue`
- `@unocss/vite`
- `@elegant-router/vue`
- `vite-plugin-vue-devtools`
- `simple-git-hooks`

它们不一定直接进入最终运行时代码，但对开发和构建非常重要。

### 一个判断方法

问自己两个问题：

1. 浏览器运行页面时是否需要它？
2. 只是开发、构建、检查、生成代码时才需要它？

如果是第一个，通常是 `dependencies`。

如果是第二个，通常是 `devDependencies`。

注意：实际项目中也会因为构建工具、包管理习惯或框架要求存在一些边界情况，不必死记，重点理解用途。

## 十五、如何阅读一个项目的技术栈

以后看到一个新项目，可以按这个顺序读。

### 第一步：看 package.json 的描述和 scripts

关注：

- 项目是什么类型。
- 用什么命令启动。
- 用什么命令构建。
- 是否有类型检查和 lint。

### 第二步：看 dependencies

判断运行时技术：

- 框架是什么。

## 十八、关键认知

### 认知 1：技术栈不是依赖列表

依赖列表只是名字，技术栈是职责分工。

你要能说出：

- Vue 负责页面。
- Vite 负责开发和构建。
- TypeScript 负责类型。
- Router 负责页面访问。
- Pinia 负责全局状态。
- Element Plus 负责 UI 组件。
- UnoCSS 负责样式效率。
- 请求封装负责接口。
- i18n 负责多语言。

### 认知 2：后台项目的核心主线只有几条

第一阶段最重要的是掌握：

- Vue 组件。
- Router 权限。
- Pinia 状态。
- 请求封装。
- Element Plus 表格表单。
- UnoCSS / SCSS 样式。

图表、编辑器、打印、地图、甘特图等插件可以后学。

### 认知 3：先学框架协作，再学单个库细节

后台项目的难点不在于某个 API，而在于多个技术如何协作。

例如登录流程会同时涉及：

- Vue 页面。
- Element Plus 表单。
- service 请求。
- Pinia auth store。
- localStorage。
- Vue Router 跳转。
- NProgress。
- i18n 提示。

所以学习时要顺着业务链路看技术配合。

### 认知 4：dependencies 和 devDependencies 不只是分类

它们能帮助你判断：

- 哪些库和业务运行有关。
- 哪些库和开发工具有关。
- 哪些问题应该在浏览器运行时排查。
- 哪些问题应该在构建或类型检查阶段排查。

### 认知 5：配置文件是理解技术栈的地图

遇到不懂的能力时，不要只在页面里找。

例如：

- 路径别名看 `vite.config.ts`。
- 原子类快捷方式看 `uno.config.ts`。
- 类型范围看 `tsconfig.json`。
- 代码规范看 `eslint.config.js`。
- workspace 看 `pnpm-workspace.yaml`。
