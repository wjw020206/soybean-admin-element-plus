# SoybeanAdmin ElementPlus 学习指南

这份指南面向想通过本仓库深入学习后台管理系统开发，并最终能独立搭建一个后台管理项目的开发者。

本项目适合作为学习样板的原因是：它覆盖了后台管理系统的核心能力，包括登录认证、权限路由、动态菜单、请求封装、Pinia 状态管理、Element Plus 组件实践、主题配置、国际化、标签页、页面缓存和典型 CRUD 页面。

## 一、项目技术栈

先从整体技术栈建立认知：

- Vue 3：应用主体框架
- Vite 7 / rolldown-vite：开发服务器与构建工具
- TypeScript：类型系统
- Vue Router：路由管理
- Pinia：全局状态管理
- Element Plus：后台 UI 组件库
- UnoCSS：原子化 CSS
- pnpm workspace：monorepo 包管理
- Axios / Alova：请求封装方案
- vue-i18n：国际化
- Elegant Router：文件路由与路由类型生成

学习时不要一开始就追求理解所有细节。后台项目的主线应该是：启动应用、登录、请求接口、生成路由、展示菜单、进入页面、完成业务 CRUD。

## 二、推荐学习顺序

### 1. 跑起来并理解启动链路

重点文件：

- `package.json`
- `vite.config.ts`
- `.env`
- `.env.test`
- `src/main.ts`
- `src/plugins/index.ts`
- `src/App.vue`

需要理解的问题：

- 项目用什么命令启动？
- Vite 如何读取环境变量？
- 应用标题、接口地址、路由模式从哪里配置？
- Vue 应用在哪里创建？
- Element Plus、Pinia、Router、i18n 分别在哪里注册？
- `setupApp` 的执行顺序为什么这样安排？

建议练习：

- 修改 `VITE_APP_TITLE`，观察页面标题变化。
- 修改 `vite.config.ts` 中的开发端口。
- 修改 `.env.test` 中的接口地址，理解请求 baseURL 的来源。
- 在 `src/main.ts` 中给每个 setup 步骤加断点或日志，观察启动顺序。

### 2. 理解登录、Token 和请求封装

重点文件：

- `src/store/modules/auth/index.ts`
- `src/store/modules/auth/shared.ts`
- `src/service/request/index.ts`
- `src/service/request/shared.ts`
- `src/service/api/auth.ts`
- `src/utils/storage.ts`

需要理解的问题：

- 登录接口在哪里调用？
- 登录成功后 token 存在哪里？
- 每次请求如何自动携带 `Authorization`？
- 后端响应什么 code 表示成功？
- 后端返回 token 过期 code 时如何刷新 token？
- 哪些错误会触发退出登录？
- 退出登录时清理了哪些本地状态？

建议练习：

- 新增一个 `fetchGetProfile` 接口。
- 在任意页面调用该接口并展示数据。
- 手动清除 localStorage 中的 token，观察路由跳转。
- 修改 `.env` 中的成功 code 或过期 code，理解请求层和后端协议的关系。

### 3. 吃透路由、菜单和权限

重点文件：

- `src/router/index.ts`
- `src/router/guard/index.ts`
- `src/router/guard/route.ts`
- `src/router/routes/index.ts`
- `src/router/routes/builtin.ts`
- `src/store/modules/route/index.ts`
- `src/store/modules/route/shared.ts`
- `src/router/elegant/routes.ts`
- `src/router/elegant/transform.ts`

需要理解的问题：

- 初始化时为什么只注册内置路由？
- 常量路由和权限路由有什么区别？
- 未登录访问受保护页面时如何跳转登录页？
- 登录后如何初始化用户信息？
- 静态权限路由如何根据角色过滤？
- 动态权限路由如何从后端获取？
- 菜单是如何从路由树生成的？
- 403、404、首页重定向分别在哪里处理？

建议先学习 `static` 模式，再学习 `dynamic` 模式。

静态模式关注：

- 前端本地维护路由。
- 根据用户角色过滤可访问路由。
- 菜单从过滤后的路由生成。

动态模式关注：

- 后端返回当前用户可访问的路由。
- 前端把后端路由转换为 Vue Router 可用的路由。
- 首页和权限判断依赖后端数据。

建议练习：

- 新增一个“订单管理”页面。
- 给页面配置标题、图标、排序。
- 让它出现在侧边菜单中。
- 给它加角色权限，验证不同角色是否能访问。
- 修改 `VITE_AUTH_ROUTE_MODE`，对比静态路由和动态路由的差异。

### 4. 拆解一个典型 CRUD 页面

重点文件：

- `src/views/manage/user/index.vue`
- `src/views/manage/user/modules/user-search.vue`
- `src/views/manage/user/modules/user-operate-drawer.vue`
- `src/service/api/system-manage.ts`
- `src/hooks/common/table`
- `src/components/advanced/table-header-operation.vue`
- `src/components/advanced/table-column-setting.vue`

用户管理页是学习 CRUD 的最佳入口，它包含后台系统中最常见的页面结构：

- 搜索表单
- 表格
- 分页
- 新增
- 编辑
- 删除
- 批量删除
- loading 状态
- 表格列配置
- 抽屉表单
- API 数据转换

需要理解的问题：

- 搜索条件如何维护？
- 分页参数如何同步到接口请求？
- 表格列是如何定义的？
- 操作列为什么使用 TSX？
- 新增和编辑为什么共用一个抽屉？
- 删除成功后为什么要刷新列表？
- `useUIPaginatedTable` 封装了哪些重复逻辑？
- `useTableOperate` 管理了哪些操作状态？

建议练习：

- 照着用户管理页实现一个“商品管理”或“订单管理”模块。
- 搜索表单至少包含 3 个字段。
- 表格至少包含 6 列。
- 实现新增、编辑、删除、分页、重置搜索。
- 把 mock 接口替换为你自己的后端接口。

### 5. 理解布局、菜单、标签页和页面缓存

重点文件：

- `src/layouts/base-layout/index.vue`
- `src/layouts/modules/global-header/index.vue`
- `src/layouts/modules/global-sider/index.vue`
- `src/layouts/modules/global-menu/index.vue`
- `src/layouts/modules/global-tab/index.vue`
- `src/layouts/modules/global-breadcrumb/index.vue`
- `src/layouts/modules/theme-drawer/index.vue`
- `src/store/modules/app/index.ts`
- `src/store/modules/tab/index.ts`
- `src/store/modules/theme/index.ts`

需要理解的问题：

- 基础布局由哪些模块组成？
- 侧边栏折叠状态在哪里维护？
- 菜单选中状态如何和当前路由同步？
- 面包屑如何从路由和菜单生成？
- 标签页如何记录访问过的页面？
- 页面缓存依赖哪些路由配置？
- 主题配置如何影响全局样式？
- 移动端布局如何适配？

建议练习：

- 修改 logo 和系统名称。
- 调整侧边栏宽度。
- 新增一个顶部工具按钮。
- 关闭或调整页签行为。
- 给某个页面开启或关闭缓存，观察页面状态是否保留。

### 6. 学习主题、样式和组件规范

重点文件：

- `src/theme/settings.ts`
- `src/theme/vars.ts`
- `src/styles/scss/global.scss`
- `src/styles/scss/element-plus.scss`
- `uno.config.ts`
- `src/components/common`
- `src/components/custom`
- `src/components/advanced`

需要理解的问题：

- 主题变量如何定义？
- Element Plus 样式如何覆盖？
- UnoCSS 在模板中如何使用？
- 公共组件按什么维度拆分？
- 哪些组件属于业务无关组件？
- 哪些组件属于后台场景增强组件？

建议练习：

- 修改主题主色。
- 新增一个通用图标按钮组件。
- 新增一个业务状态标签组件。
- 给已有页面统一调整间距和视觉层级。

### 7. 最后再看 monorepo 和工程化

重点目录：

- `packages/axios`
- `packages/alova`
- `packages/hooks`
- `packages/utils`
- `packages/uno-preset`
- `packages/scripts`
- `pnpm-workspace.yaml`
- `eslint.config.js`
- `tsconfig.json`

需要理解的问题：

- 为什么公共能力放在 `packages` 中？
- workspace 包如何被主应用引用？
- `@sa/hooks`、`@sa/axios`、`@sa/utils` 分别提供什么？
- ESLint、TypeScript、git hooks 如何保证代码质量？
- `pnpm typecheck` 和 `pnpm lint` 分别检查什么？

这一部分建议放在后期学习。初期重点应该是业务链路，而不是工程化细节。

## 三、四周学习计划

### 第 1 周：熟悉项目和启动链路

目标：

- 能独立启动项目。
- 能说明 `src/main.ts` 的初始化过程。
- 能修改环境变量并理解其影响。
- 能找到应用入口、路由入口、状态入口、请求入口。

产出：

- 一张“项目启动流程图”。
- 一份“关键目录说明笔记”。

### 第 2 周：登录、请求和权限路由

目标：

- 能解释登录流程。
- 能解释 token 存储和请求拦截。
- 能解释路由守卫的判断顺序。
- 能新增一个静态权限页面。

产出：

- 一张“登录到进入首页流程图”。
- 一个新增的测试页面，并出现在菜单中。

### 第 3 周：完成一个 CRUD 模块

目标：

- 能照着用户管理实现一个新业务模块。
- 能写搜索表单、表格、分页、抽屉表单。
- 能对接接口并处理 loading 和错误状态。

产出：

- 一个“订单管理”或“商品管理”模块。
- 至少包含列表、搜索、新增、编辑、删除功能。

### 第 4 周：脱离模板搭建简化版后台

目标：

- 从零创建一个 Vite + Vue + TypeScript + Element Plus 项目。
- 迁移本项目中你真正理解的模块。
- 实现一个最小可用后台系统。

产出：

- 登录页
- 首页
- 左侧菜单
- 权限路由
- 请求封装
- 用户管理或订单管理
- 403/404 页面

## 四、最终实战目标

学习这个仓库的最终目标不是“看懂源码”，而是能独立完成一个自己的后台管理项目。

建议你最终实现一个小型后台，包含：

- 登录页
- 首页仪表盘
- 用户管理
- 角色管理
- 菜单管理
- 一个业务模块，例如订单管理
- 静态权限路由
- token 请求拦截
- 403/404 页面
- 表格分页和搜索
- 新增/编辑弹窗或抽屉
- 基础主题配置

完成这些内容后，你已经具备独立搭建后台管理系统的核心能力。

## 五、自检问题

每学完一个阶段，用下面的问题检查自己是否真正掌握：

- 我能不能新增一个页面，并让它出现在菜单中？
- 我能不能新增一个接口，并在页面中调用？
- 我能不能解释 token 过期后的处理流程？
- 我能不能控制某个角色看不到某个菜单？
- 我能不能把用户管理页改造成订单管理页？
- 我能不能解释菜单和路由之间的关系？
- 我能不能解释为什么刷新页面后路由需要重新初始化？
- 我能不能从零搭一个简化版后台？

如果这些问题都能答出来，说明你不是只会使用模板，而是真的理解了后台管理系统的核心设计。

## 七、学习建议

- 不要只看代码，要边改边验证。
- 不要一开始就研究所有封装，先顺着业务链路走。
- 不要直接复制页面，至少要能解释每个状态变量和每个函数的作用。
- 优先掌握静态权限路由，再研究动态权限路由。
- 优先掌握一个 CRUD 页面，再扩展到复杂业务。
- 每次改动都运行 `pnpm typecheck` 和 `pnpm lint`，养成工程化习惯。

最有效的学习方式是：阅读一个模块，画出流程图，照着实现一个类似模块，然后脱离模板再写一遍简化版。
