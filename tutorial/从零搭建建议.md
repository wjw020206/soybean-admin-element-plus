# 从零搭建后台项目的学习建议

这份文档用于补充 `课程大纲.md`。

原课程大纲中，第 29 课安排了“从零搭建一个简化版后台”。但在实际学习时，不建议等到第 29 课才第一次动手从零搭建。更好的方式是：前期理解项目，中期分阶段搭建，后期完整复盘。

## 一、为什么不要等到第 29 课才动手

第 1-4 课主要是认知和地图：

- 第 1 课：后台管理系统到底由什么组成
- 第 2 课：开发环境与项目启动
- 第 3 课：技术栈全景图
- 第 4 课：目录结构与源码阅读方法

这些内容偏概念，但不是无用概念。它们帮助你知道：

- 项目是什么。
- 项目怎么跑。
- 技术怎么分工。
- 源码应该怎么看。

如果没有这些地图，后面直接写代码很容易变成复制粘贴。

但如果一直学概念，不动手，也会出现另一个问题：你觉得自己看懂了，但真正从零建项目时不知道第一步做什么。

所以建议采用“分阶段从零搭建”的方式。

## 二、推荐的分阶段搭建节奏

可以把“从零搭建后台”拆成四次小实战和一次最终实战。

| 时间点 | 可以从零搭建的内容 | 目标 |
|---|---|---|
| 第 8 课后 | 最小后台骨架 | 搭出 Vue + Vite + TS + Router + Pinia + Element Plus |
| 第 13 课后 | 登录和请求骨架 | 加入登录页、token、请求封装、用户信息 |
| 第 18 课后 | 权限后台骨架 | 加入路由守卫、菜单、403、404、静态权限 |
| 第 23 课后 | 业务后台模块 | 加入表格、搜索、分页、新增、编辑、删除 |
| 第 29 课 | 完整复盘式从零搭建 | 独立搭建一个完整 mini-admin |

这样做的好处是：

- 每学完一个阶段都有产出。
- 不会到最后才发现自己不会搭。
- 每次搭建目标都足够小，容易完成。
- 最终第 29 课不是第一次动手，而是系统复盘。

## 三、第 8 课后：搭建最小后台骨架

完成第 5-8 课后，你已经学习了：

- `src/main.ts` 启动链路。
- Vite 配置和环境变量。
- 插件系统和全局能力注册。
- TypeScript 类型体系和路径别名。

这时可以从零搭一个最小后台骨架。

目标功能：

- 创建 Vite + Vue + TypeScript 项目。
- 安装 Element Plus。
- 安装 Vue Router。
- 安装 Pinia。
- 配置 `@` 路径别名。
- 创建基础 Layout。
- 创建首页。
- 创建 404 页面。
- 能正常启动项目。

建议目录：

```txt
mini-admin
├── src
│   ├── main.ts
│   ├── App.vue
│   ├── router
│   ├── store
│   ├── layouts
│   ├── views
│   └── styles
├── vite.config.ts
├── tsconfig.json
└── package.json
```

验收标准：

- 执行 `pnpm dev` 能打开页面。
- 首页能通过路由访问。
- Element Plus 组件能正常使用。
- Pinia store 能在页面中读取和修改状态。
- `@/xxx` 路径别名可用。

这一阶段不要急着做登录和权限，目标只是搭出应用骨架。

## 四、第 13 课后：加入登录和请求封装

完成第 9-13 课后，你已经学习了：

- Pinia 状态管理。
- 登录流程。
- 本地存储。
- 请求封装。
- Token 过期和刷新机制。

这时可以给 mini-admin 加入登录和请求能力。

目标功能：

- 登录页。
- 登录接口。
- token 存储。
- 用户信息接口。
- 请求前自动携带 token。
- 请求错误统一提示。
- 退出登录。
- token 失效后回到登录页。

建议新增目录：

```txt
src
├── service
│   ├── request
│   └── api
├── utils
│   └── storage.ts
└── store
    └── modules
        └── auth.ts
```

验收标准：

- 未登录时只能访问登录页。
- 登录成功后 token 写入本地存储。
- 刷新页面后能根据 token 保持登录状态。
- 请求接口时能自动带上 Authorization。
- 退出登录后清理 token 和用户信息。

这一阶段可以先使用 mock 数据，不必急着接真实后端。

## 五、第 18 课后：加入权限路由和菜单

完成第 14-18 课后，你已经学习了：

- Vue Router 初始化。
- 路由守卫。
- 静态权限路由。
- 动态权限路由。
- 菜单、面包屑、缓存页和标签页的关系。

这时可以给 mini-admin 加入权限后台骨架。

目标功能：

- 基础路由。
- 登录白名单。
- 路由守卫。
- 静态权限路由。
- 根据路由生成菜单。
- 根据角色过滤菜单。
- 403 页面。
- 404 页面。
- 面包屑。

建议新增目录：

```txt
src
├── router
│   ├── index.ts
│   ├── guard.ts
│   └── routes.ts
├── layouts
│   └── modules
│       ├── sidebar
│       ├── header
│       └── breadcrumb
└── store
    └── modules
        └── route.ts
```

验收标准：

- 未登录访问受保护页面会跳转登录页。
- 登录后根据角色展示不同菜单。
- 无权限访问页面时跳转 403。
- 不存在的页面跳转 404。
- 菜单点击能切换页面。
- 面包屑能根据当前路由变化。

这一阶段先做静态权限路由即可。动态权限路由可以作为进阶任务。

## 六、第 23 课后：加入业务 CRUD 模块

完成第 19-23 课后，你已经学习了：

- 用户管理页面结构。
- 搜索表单。
- 表格和分页。
- 列配置。
- 新增、编辑、删除。
- 抽屉表单。
- 订单管理实战。

这时可以给 mini-admin 加入一个完整业务模块。

推荐模块：

- 用户管理。
- 商品管理。
- 订单管理。
- 角色管理。

目标功能：

- 搜索表单。
- 数据表格。
- 分页。
- 新增。
- 编辑。
- 删除。
- 状态标签。
- 表单校验。
- loading 状态。
- 接口请求。

建议目录：

```txt
src
└── views
    └── order
        ├── index.vue
        └── modules
            ├── order-search.vue
            └── order-operate-drawer.vue
```

验收标准：

- 页面能通过菜单进入。
- 搜索条件能影响列表。
- 分页能正常切换。
- 新增成功后刷新列表。
- 编辑能回显当前行数据。
- 删除前有确认。
- 表单必填项有校验。

这一阶段是从“后台骨架”进入“业务后台”的关键。

## 七、第 29 课：完整复盘式从零搭建

到了第 29 课，再进行一次完整从零搭建。

这一次的目标不是第一次尝试，而是把前面分阶段学过的内容完整串起来。

最终 mini-admin 应该包含：

- 登录页。
- 首页。
- 基础 Layout。
- 侧边菜单。
- 顶部栏。
- 面包屑。
- 路由守卫。
- 静态权限路由。
- token 请求拦截。
- 用户信息 store。
- 403 页面。
- 404 页面。
- 一个完整 CRUD 模块。
- 基础主题样式。
- 类型检查和 lint。

最终验收标准：

- 从空项目开始，不复制整套模板。
- 能解释每个目录为什么存在。
- 能解释登录流程。
- 能解释路由守卫判断顺序。
- 能解释菜单如何生成。
- 能解释请求如何携带 token。
- 能独立新增一个业务页面。

## 八、建议调整课程节奏

如果希望课程更偏实战，可以在原 30 课中加入阶段性实战任务。

建议方式：

```txt
第 1-4 课：概念和源码地图
第 5-8 课：应用骨架
第 8 课后：实战 A，搭建最小后台骨架

第 9-13 课：登录、请求、状态
第 13 课后：实战 B，加入登录和请求封装

第 14-18 课：路由、菜单、权限
第 18 课后：实战 C，加入权限路由和菜单

第 19-23 课：CRUD 页面
第 23 课后：实战 D，完成订单管理模块

第 24-28 课：布局、主题、国际化、工程化、真实后端
第 29 课：完整从零搭建 mini-admin
第 30 课：复盘和验收
```

## 九、学习建议

- 第 1-4 课不要急着写完整项目，先建立地图。
- 第 5 课开始就可以做小实验，例如自己写一个 `main.ts` 启动流程。
- 第 8 课后必须动手搭一次最小骨架。
- 第 13 课后必须动手写一次登录和请求封装。
- 第 18 课后必须动手写一次权限路由。
- 第 23 课后必须动手写一个完整 CRUD。
- 第 29 课不要照抄模板，要从空项目开始搭。

真正掌握后台项目的标志不是能看懂源码，而是能在空项目中按需搭出这些能力。
