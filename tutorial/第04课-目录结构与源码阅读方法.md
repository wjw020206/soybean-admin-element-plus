# 第 4 课：目录结构与源码阅读方法

## 课程定位

第 1 课到第 3 课分别建立了后台系统认知、开发环境认知和技术栈认知。第 4 课要建立的是源码目录地图。

这一课仍然不要求逐行阅读源码。重点是知道每个目录负责什么、后续遇到问题应该去哪里找、学习时应该按什么顺序深入，而不是打开一个文件就一路追到最底层。

很多人学习开源后台项目时容易犯一个错误：一上来就从某个复杂文件开始追调用链，比如从路由守卫追到 route store，再追到接口，再追到类型，再追到工具函数，最后迷路。正确方式是先看目录职责，再按业务链路分阶段深入。

## 学习目标

学完本课后，你应该能够：

- 说清楚 `src` 下主要目录的职责。
- 说清楚 `packages` 下公共包的大致作用。
- 知道哪些目录是前期必须掌握的，哪些目录可以后期再看。
- 建立后台项目的源码阅读顺序。
- 能够画出一张项目目录结构图。
- 能够在遇到问题时快速定位相关目录。

## 建议课时

建议用 60-90 分钟完成本课：

- 15 分钟：浏览 `src` 一级目录。
- 20 分钟：理解路由、状态、请求、页面、布局这些核心目录。
- 15 分钟：理解组件、hooks、主题、国际化、类型目录。
- 15 分钟：浏览 `packages` 公共包。
- 15 分钟：整理目录职责图和源码阅读路线。

## 一、本课的阅读原则

第 4 课只做“目录级阅读”，不要深入每个文件的实现细节。

你要做到的是：

- 能说出目录职责。
- 能知道关键入口文件。
- 能知道后续哪一课会深入它。
- 能根据问题定位目录。

你不需要现在就做到：

- 看懂所有路由守卫逻辑。
- 看懂权限路由生成细节。
- 看懂请求封装内部实现。
- 看懂所有 hooks。
- 看懂所有公共包源码。

这一课看的是地图，不是地形。

## 二、项目根目录先看什么

项目根目录包含工程配置、依赖管理和应用源码。

重点文件和目录：

```txt
soybean-admin-element-plus
├── package.json
├── pnpm-workspace.yaml
├── pnpm-lock.yaml
├── vite.config.ts
├── uno.config.ts
├── tsconfig.json
├── eslint.config.js
├── index.html
├── public
├── build
├── packages
└── src
```

### 1. `package.json`

职责：

- 定义项目名称、版本、依赖、脚本命令。
- 告诉你项目如何启动、构建、检查和提交。

学习阶段：

- 第 2 课已经浅看。
- 后续每次遇到命令或依赖问题都要回来看。

### 2. `vite.config.ts`

职责：

- 配置 Vite 开发服务器和构建。
- 配置路径别名。
- 配置代理。
- 注册 Vite 插件。

学习阶段：

- 第 2、3 课已经浅看。
- 第 6 课会深入。

### 3. `uno.config.ts`

职责：

- 配置 UnoCSS。
- 配置主题 token、快捷类、预设、转换器。

学习阶段：

- 第 3 课已经浅看。
- 第 25 课主题和样式体系会深入。

### 4. `packages`

职责：

- 存放 monorepo 公共包。
- 提供请求、hooks、工具函数、脚本、UnoCSS 预设等能力。

学习阶段：

- 前期只知道它存在。
- 第 27 课再深入。

### 5. `src`

职责：

- 应用主源码。
- 本课程主要学习对象。

学习阶段：

- 从第 5 课开始逐步深入。

## 四、`src/router`：路由体系

目录结构：

```txt
src/router
├── index.ts
├── guard
│   ├── index.ts
│   ├── progress.ts
│   ├── route.ts
│   └── title.ts
├── routes
│   ├── builtin.ts
│   └── index.ts
└── elegant
    ├── imports.ts
    ├── routes.ts
    └── transform.ts
```

职责：

- 创建 Vue Router 实例。
- 注册路由守卫。
- 管理内置路由、常量路由、权限路由。
- 处理静态路由和动态路由。
- 将 Elegant Router 路由转换为 Vue Router 路由。

关键文件：

- `src/router/index.ts`：路由实例入口。
- `src/router/guard/route.ts`：访问控制主逻辑。
- `src/router/routes/builtin.ts`：内置路由。
- `src/router/routes/index.ts`：自定义路由和静态路由创建。
- `src/router/elegant/routes.ts`：生成路由。
- `src/router/elegant/transform.ts`：路由转换。

现在要知道：

- `src/router` 是后台权限和菜单体系的核心目录之一。
- 后续第 14-18 课会深入。

现在不要深入：

- 不要急着读完整 `route.ts`。
- 不要急着追动态路由全部流程。
- 不要急着研究 Elegant Router 所有类型。

## 五、`src/store`：状态管理

目录结构：

```txt
src/store
├── index.ts
├── plugins
│   └── index.ts
└── modules
    ├── app
    ├── auth
    ├── route
    ├── tab
    └── theme
```

职责：

- 注册 Pinia。
- 管理应用全局状态。
- 保存认证信息、路由菜单、标签页、主题和应用布局状态。

核心模块：

- `auth`：登录认证、token、用户信息。
- `route`：权限路由、菜单、面包屑、缓存路由。
- `tab`：多标签页。
- `theme`：主题、暗色模式、布局配置。
- `app`：应用级状态。

关键文件：

- `src/store/index.ts`：Pinia 注册入口。
- `src/store/modules/auth/index.ts`：认证状态。
- `src/store/modules/route/index.ts`：路由菜单状态。
- `src/store/modules/tab/index.ts`：标签页状态。
- `src/store/modules/theme/index.ts`：主题状态。

现在要知道：

- store 不是随便存数据的地方。
- 只有跨页面、跨组件共享的状态才适合放进 store。

后续深入：

- 第 9 课讲 Pinia。
- 第 10-13 课讲 auth。
- 第 14-18 课讲 route。
- 第 24-25 课讲 tab 和 theme。

## 六、`src/service` 和 `src/service-alova`：接口请求

目录结构：

```txt
src/service
├── api
│   ├── auth.ts
│   ├── index.ts
│   ├── route.ts
│   └── system-manage.ts
└── request
    ├── index.ts
    ├── shared.ts
    └── type.ts
```

```txt
src/service-alova
├── api
├── mocks
└── request
```

职责：

- 封装请求实例。
- 管理 API 方法。
- 处理 token、错误、后端 code、刷新 token。
- 区分不同请求方案。

`src/service` 和 `src/service-alova` 的关系：

- `src/service` 是一套基于项目 axios 封装的请求方案。
- `src/service-alova` 是另一套基于 alova 的请求方案和示例。
- 学习时先掌握 `src/service`，再对比 `service-alova`。

关键文件：

- `src/service/request/index.ts`：请求实例主配置。
- `src/service/request/shared.ts`：请求工具函数。
- `src/service/api/auth.ts`：登录相关接口。
- `src/service/api/system-manage.ts`：系统管理接口。

现在要知道：

- 页面不应该直接写裸请求。
- 页面应该调用 `src/service/api` 中封装好的 API 函数。

后续深入：

- 第 12-13 课。

## 七、`src/views`：业务页面

目录结构示例：

```txt
src/views
├── _builtin
├── about
├── alova
├── function
├── home
├── manage
├── multi-menu
├── plugin
└── user-center
```

职责：

- 存放页面级组件。
- 每个目录通常对应一个业务模块或路由模块。

重点目录：

- `_builtin`：内置页面，比如登录、异常页等。
- `home`：首页相关页面和模块。
- `manage`：用户、角色、菜单等系统管理页面。
- `function`：功能示例页面。
- `plugin`：插件示例页面。
- `alova`：alova 请求示例页面。

以用户管理为例：

```txt
src/views/manage/user
├── index.vue
└── modules
    ├── user-operate-drawer.vue
    └── user-search.vue
```

页面拆分思路：

- `index.vue`：页面主容器，组织搜索、表格、分页、操作。
- `modules/user-search.vue`：搜索表单。
- `modules/user-operate-drawer.vue`：新增和编辑抽屉。

现在要知道：

- 业务页面通常不是一个 `.vue` 文件写到底。
- 复杂页面会拆成 `modules` 子组件。

后续深入：

- 第 19-23 课。

## 八、`src/layouts`：后台布局

目录结构：

```txt
src/layouts
├── base-layout
├── blank-layout

    ├── global-menu
    ├── global-search
    ├── global-sider
    ├── global-tab
    └── theme-drawer
```

职责：

- 提供后台管理系统整体页面框架。
- 管理顶部栏、侧边栏、菜单、面包屑、内容区、标签页、主题抽屉。

核心模块：

- `base-layout`：登录后主布局。
- `blank-layout`：空白布局，适合登录页、异常页等。
- `global-header`：顶部栏。
- `global-sider`：侧边栏。
- `global-menu`：菜单。

- `global-breadcrumb`：面包屑。

- 后台系统的布局是一个独立层。
- 业务页面通常渲染在布局的内容区中。
- 菜单、面包屑、标签页都和路由状态有关。

## 九、`src/components`：公共组件

目录结构：

```txt
src/components
├── advanced
├── common
└── custom
```

职责：

- 存放可复用组件。
- 减少页面重复代码。

分组含义：

### 1. `common`

偏通用的系统组件。

示例：

- `app-provider.vue`
- `dark-mode-container.vue`
- `exception-base.vue`
- `full-screen.vue`
- `lang-switch.vue`
- `system-logo.vue`
- `theme-schema-switch.vue`

### 2. `custom`

项目自定义基础组件。

示例：

- `svg-icon.vue`
- `button-icon.vue`
- `count-to.vue`
- `soybean-avatar.vue`
- `better-scroll.vue`

### 3. `advanced`

更贴近后台场景的增强组件。

示例：

- `table-column-setting.vue`
- `table-header-operation.vue`

现在要知道：

- 组件目录不是页面目录。
- 组件应该服务多个页面或多个布局模块。
- 不要为了一个页面的临时结构过早抽公共组件。

后续深入：

- 在 CRUD 页面学习中重点看 `advanced`。

## 十、`src/hooks`：组合式逻辑

目录结构：

```txt
src/hooks
├── business
│   ├── auth.ts
│   └── captcha.ts
└── common
    ├── echarts.ts
    ├── form.ts
    ├── icon.ts
    ├── router.ts
    ├── table.ts
    └── vchart.ts
```

职责：

- 把可复用逻辑从组件中抽离出来。
- 让页面组件更专注于结构和业务编排。

分组含义：

- `business`：偏业务的 hooks，比如权限、验证码。
- `common`：偏通用能力，比如表格、表单、路由、图表。

关键文件：

- `src/hooks/common/table.ts`：表格分页、列设置、操作状态等。
- `src/hooks/common/router.ts`：路由跳转封装。
- `src/hooks/business/auth.ts`：权限相关逻辑。

现在要知道：

- hooks 是复用逻辑，不是页面。
- 看到页面中调用 `useXxx`，可以到 `src/hooks` 找实现。

后续深入：

- 第 19-23 课会结合 CRUD 页面学习 `table.ts`。

## 十五、`src/constants`、`src/enum`、`src/utils`：常量、枚举和工具函数

### `src/constants`

职责：

- 存放稳定常量和业务映射。

示例：

- `app.ts`：应用常量。
- `business.ts`：业务字典。
- `common.ts`：通用常量。
- `reg.ts`：正则。

### `src/enum`

职责：

- 存放枚举类常量。
- 比如 store id 等固定枚举值。

### `src/utils`

职责：

- 存放纯工具函数。

示例：

- `storage.ts`：本地存储封装。
- `service.ts`：接口地址处理。
- `icon.ts`：图标处理。
- `common.ts`：通用工具。

现在要知道：

- constants 偏“数据常量”。
- enum 偏“固定枚举”。
- utils 偏“函数工具”。

后续深入：

- 会随着业务模块逐步遇到。

## 十六、`src/assets` 和 `public`：静态资源

### `src/assets`

目录：

```txt
src/assets
├── imgs
└── svg-icon
```

职责：

- 存放参与构建处理的图片和 SVG 图标。
- 可以通过 import 引用。

### `public`

职责：

- 存放静态公开资源。
- 构建时通常原样复制到输出目录。

区别：

- `src/assets` 适合被源码 import。
- `public` 适合直接通过 URL 访问。

## 十七、`packages`：monorepo 公共包

当前项目有：

```txt
packages
├── alova
├── axios
├── color
├── hooks
├── materials
├── ofetch
├── scripts
├── uno-preset
└── utils
```

职责：

- 抽取跨项目复用能力。
- 供主应用通过 `workspace:*` 使用。

大致作用：

- `axios`：Axios 请求封装。
- `alova`：Alova 请求封装。
- `hooks`：通用 hooks。
- `utils`：工具函数。
- `color`：颜色处理。
- `uno-preset`：UnoCSS 项目预设。
- `scripts`：命令行脚本。
- `materials`：物料。
- `ofetch`：另一种请求封装能力。

现在要知道：

- `packages` 是公共能力，不是业务页面。
- 前期不要深入公共包源码。
- 等你能写页面、读请求、读路由后，再回头看公共包会更容易。

后续深入：

- 第 27 课。

## 十九、遇到问题时如何定位目录

可以按问题类型定位。

| 问题 | 优先看哪里 |
|---|---|
| 项目启动失败 | `package.json`、`vite.config.ts`、`.env*` |
| 路由跳转异常 | `src/router` |
| 未登录跳转、403、404 | `src/router/guard`、`src/store/modules/auth` |
| 菜单不显示 | `src/store/modules/route`、`src/layouts/modules/global-menu` |
| 接口地址不对 | `.env*`、`src/utils/service.ts`、`src/service/request` |
| 请求没带 token | `src/service/request`、`src/store/modules/auth` |
| 用户信息不对 | `src/store/modules/auth`、`src/service/api/auth.ts` |
| 表格分页不对 | `src/views/*`、`src/hooks/common/table.ts` |
| Element Plus 样式不对 | `src/styles/scss/element-plus.scss`、`src/plugins/ui.ts` |
| 暗色模式不对 | `src/store/modules/theme`、`src/theme`、`uno.config.ts` |
| 文案不对 | `src/locales/langs` |
| 类型报错 | `src/typings`、业务目录 `type.ts` |
| 公共函数找不到 | `src/utils`、`packages/utils` |

这张表比无序搜索更重要。
